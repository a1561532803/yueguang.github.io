简答题

什么是docker？

答：docker是一个容器平台，可以将开发的项目连同环境一起打包，上线部署。

Docker与虚拟机的不同点在哪里？

答：

介绍：

​ docker是一个容器平台，将应用打包进docker后，启动的容器使用沙箱机制，相互隔离，互不影响。

​ 虚拟机是运行在操作系统中的一个虚拟系统。

区别：

​ 1.启动时间：容器时秒级启动，虚拟机是分钟级启动。

​ 2.轻量级：容器一般是m级别的大小，虚拟机时g级别的大小。

​ 3.性能：容器内部的系统仅支持系统的正常运行，虚拟机则需要一套完整的系统。

Docker容器有几种状态，分别是什么？

答：四种状态；运行，已暂停，重新启动，退出

拓展：docker容器的生命周期？

​ 答：一个docker容器从产生到运行部署大致分为如下三个状态：Dockerfile、Image、Contaniner。

​ 即一个镜像文件从产生、运行到停止的过程。

写出Dockerfile中最常见四条指令，并指明含义？

答：

​ From 指定基础镜像

​ RUN 运行指定命令

​ CMD 容器启动时运行的命令

​ COPY 拷贝文件到镜像

​ ADD 同copy，但多了解压缩的功能

​ VOLUME 指定挂载点，容器和宿主机的目录

​ EVN 设定容器内环境变量

Docker镜像特点是什么？如何自定义镜像？

答：（1）Docker镜像就是一个只读的模板，容器在启动的时候创建一层可写层作为最上层

（2）方法一：使用DockerFile直接创建自定义镜像，基本步骤分别为：编写Dockerfile文件、docker build 生成镜像、docker run 创建容器运行。

方法二：使用IDEA创建自定义镜像

拓展：启动后，可通过日志查询命令查看是否已启动完毕且成功。

docker logs -f 镜像名

Docker 数据卷有什么作用？创建数据卷的命令是？

答：容器的持久化、容器间继承与共享数据。

命令：

docker run -it -v /宿主机绝对路径目录:/容器内目录      镜像名

详细补充：

​ Docker数据卷作用（特点）：

1. 数据卷可在容器之间共享或重用数据
 2. 卷中的更改可以直接生效
 3. 数据卷中的更改不会包含在镜像的更新中
 4. 数据卷的生命周期一直持续到没有容器使用它为止

创建数据卷的命令：

使用 -v 命令，也可以在Dockerfile中使用VOLUME指令来给镜像添加一个或多个数据卷

-v命令

docker run -it -v /宿主机绝对路径目录:/容器内目录      镜像名

带权限的-v命令： 

docker run -it -v /宿主机绝对路径目录:/容器内目录:ro 镜像名`

说明：

出于可移植和分享的考虑，用-v 主机目录:容器目录这种方法不能够直接在Dockerfile中实现。

由于宿主机目录是依赖于特定宿主机的，并不能够保证在所有的宿主机上都存在这样的特定目录。

使用file构建

重点：

VOLUME["/dataVolumeContainer","/dataVolumeContainer2","/dataVolumeContainer3"]
# volume test

FROM centos

VOLUME ["/dataVolumeContainer1","/dataVolumeContainer2"]

CMD echo "finished,--------success1"

CMD /bin/bash

Docker仓库有哪些，它们的区别是什么？

答：分公有仓库与私有仓库

公有仓库：Docker Hub、时速云镜像仓库、网易云镜像服务、DaoCloud镜像市场、阿里云镜像库等

私有仓库：用户自己在本地搭建私有Docker Register（最常用的就是registry、Harbor两种）

区别：公有仓库所有人都可以搜索和下载公共镜像，私有仓库只有私有用户才能查询和下载。

什么是Docker Swarm，其功能是什么？

答：它是Docker的本机群集（Swarm 是目前 Docker 官方唯一指定（绑定）的集群管理工具）；它将Docker主机池转变为单个虚拟Docker主机（它可以把多个 Docker 主机组成的系统转换为单一的虚拟 Docker 主机），Docker Swarm提供标准的Docker API，使得容器可以组成跨主机的子网网络。

拓展：Docker三剑客

Docker Machine、Docker Compose、Docker Swarm 并称 Docker 三剑客。

什么是Docker Compose，使用的步骤是什么?

答：（1）Docker Compose是一个用来定义和运行复杂应用的Docker工具。

使用Docker Compose不再需要使用shell脚本来启动容器，而使用服务编排的方式来管理容器。

Docker Compose核心是通过一个YAML文件来管理多个Docker容器，接着就可以从配置中创建并启动所有服务。

（2）①定义要创建的应用，②创建容器的Dockerfile文件，③定义docker-compose脚本（创建docker-compose.yml文件，并在里面定义服务），④使用Compose构建并运行应用程序，⑤编辑compose文件以添加文件绑定挂载，⑥使用更新的compose文件重新构建和运行应用程序。

脚本文件示例：

version: '3'
services:
  web:
    build: ../
    ports:
     - "5000:5000"
    volumes:
     - ../src:/opt/src
  redis:
    image: "redis:3.0.7"

上面这个compose文件就定义了两个服务，即定义了web和redis两个容器。

再通过volumes（卷）将主机上的项目目录（../src）挂载到容器中的/opt/src目录下，允许您即时修改代码，而无需重新构建映像。

web容器：

  • 使用当前docker-compose.yml文件所在目录的上级目录（compose_test/Dockerfile）中的Dockerfile构建映像。
  • 将容器上的暴露端口5000映射到主机上的端口5000。 我们使用Flask Web服务器的默认端口5000。
    redis容器：
  • redis服务使用从Docker Hub提取的官方redis镜像3.0.7版本。

最后一步命令：

docker-compose up -d

Registry服务端、客户端如何配置？

本题答案参考自https://blog.51cto.com/u_14306186/2514802

答：准备两台docker服务器，dockerA创建registry私有仓库，dockerB做测试的客户端。

tips：本题主要记配置文件的代码

即docker A：ExecStart=/usr/bin/dockerd --insecure-registry 服务器IP地址:5000端口号

docker B： ExecStart=/usr/bin/dockerd --insecure-registry 客户端IP地址:5000端口号

注：客户端一定要和服务器仓库ip一致 不然无法识别和上传 至于5000端口 应为registry默认5000端口

详细步骤：

docker A:

[root@dockerA ~]# docker pull registry:2              //下载registry:2的镜像
[root@dockerA ~]# docker run -itd --name registry --restart=always  -p 5000:5000 -v /registry:/var/lib/registry registry:2
//创建一个registry容器来运行registry服务；
//-p：端口映射（前面是宿主机端口：后面是容器暴露的端口）；
//-v：挂载目录（前面是宿主机的目录：后面的是容器的目录）自动创建宿主机的目录；
//--restart=always：随docker服务的启动而启动！

[root@dockerA ~]# docker tag  centos:7 192.168.45.129:5000/centos:7  
//更改镜像名称
注：私有仓库镜像的命名规则：192.168.45.129:5000/XXX（宿主机的IP:5000端口/镜像名称）

[root@dockerA ~]# vim /usr/lib/systemd/system/docker.service   //修改docker主配置文件
ExecStart=/usr/bin/dockerd    --insecure-registry 192.168.45.129:5000  
//修改原本的配置文件添加不安全的仓库（--insecure-registry），地址是宿主机的IP地址与5000端口
[root@dockerA ~]# systemctl daemon-reload 
[root@dockerA ~]# systemctl restart docker             //重新启动docker服务
[root@dockerA ~]# docker push  192.168.45.129:5000/centos:7
//将重命名后的镜像上传到registry私有仓库
[root@dockerA ~]# curl 192.168.45.129:5000/v2/_catalog             //查看私有仓库中的镜像
{"repositories":["centos"]}
[root@dockerA ~]# curl 192.168.45.129:5000/v2/centos/tags/list         //查看镜像的详细信息
{"name":"centos","tags":["7"]}

dockerB：

[root@dockerB ~]# vim /usr/lib/systemd/system/docker.service 
//修改docker的主配置文件
 ExecStart=/usr/bin/dockerd --insecure-registry 192.168.45.129:5000
//添加内容与registry一致，指定registry私有仓库服务器的IP地址及端口
[root@dockerB ~]# systemctl daemon-reload 
[root@dockerB ~]# systemctl restart docker            //重新启动docker服务
[root@dockerB ~]# curl 192.168.45.129:5000/v2/_catalog         //查看私有仓库中的镜像
{"repositories":["centos"]}
[root@dockerB ~]# curl 192.168.45.129:5000/v2/centos/tags/list     //查看私有仓库中的镜像
{"name":"centos","tags":["7"]}
[root@dockerB ~]# docker pull 192.168.45.129:5000/centos:7
